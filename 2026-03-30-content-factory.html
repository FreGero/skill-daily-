Content Factory — 多智能体内容工厂

一份素材进，五种格式出。五个 AI 角色协作的内容流水线，把"写一篇文章"变成"生产一个内容矩阵"。

📦 Skill 概述
一句话：一份内容 → 多平台发布，五个专业 Agent 各司其职的内容流水线。

Content Factory 是一个结构化的多智能体内容生产系统。它不是让一个 AI 干所有事，而是设计了五个专门的角色——Writer（写手）、Remixer（混剪师）、Editor（编辑）、Scriptwriter（编剧）和 Headline Machine（标题机器）——每个角色有自己的工作模板、质量标准和输出格式。


你可以跑完整流水线（从草稿到全平台内容），也可以单独调用任何一个 Agent。核心哲学很朴素：别重复造轮子，一份好内容值得在每个平台都被看见。

🧠 核心理念

Content Factory 的设计哲学建立在三个关键洞察上：

1. 角色分工 > 全能选手
与其让一个 AI 又写又编又排，不如让五个"专家"各管一段。Writer 负责初稿质量，Editor 负责精简打磨，Remixer 负责平台适配——每一环的 Prompt 都是为那个具体任务深度优化的。这就像一个真实的出版社编辑部，而非一个全干的自媒体人。

2. 品牌一致性是刚需
跨平台内容最怕的不是量不够，是声音不统一。Content Factory 用 Voice Cloner 和统一的内容原则确保同一条信息在 Twitter、LinkedIn、邮件里虽然形态不同，但人格一致。

3. 流程可拆可组
你可以跑全流水线，也可以只叫一个 Agent 干活。这种模块化设计意味着它不是一个"用了就得全用"的臃肿系统，而是一个灵活的工具箱。

🔑 关键功能模式
Prompt 模板 First Draft — 脑暴笔记→结构化文章

这是 Writer Agent 最常用的模板。你丢进去一堆零散笔记，它帮你组织成一篇完整文章。

结构：

  • Hook（5%） — 1-2句，用一个具体场景或问题抓住读者
  • Setup（15%） — 为什么这件事此刻对读者重要
  • Body（60%） — 3-5个小节，每节一个清晰论点，H2 子标题
  • Close（20%） — 一个明确的行动步骤或反思提示

关键指令："保留我的想法和例子——不要替换成通用的"。这一条很重要，防止 AI 把你的独特素材洗成通稿。

// 使用示例
"Act as the Writer agent. Write a 1,200-word article on [topic] 
for [target audience]. Use the first-draft template."
Prompt 模板 Remix Engine — 一篇文章→11种格式

Remixer Agent 的核心模板。给它一篇成稿，它同时输出所有平台格式。

支持的11种输出格式：

  • Twitter/X Thread（8-10条，第一条必须是停止滑动的钩子）
  • LinkedIn Post（150字，以洞察开头，以问题结尾）
  • Email Newsletter Section（含主题行+预览文本，亲密语气）
  • Slide Deck Outline（8页，每页一个想法）
  • Executive Summary（三段：What, So What, Now What）
  • FAQ Section（5个真实问题，对话式回答）
  • Pull Quotes（5条推文级金句，每条<280字符）
  • Podcast Script Intro（2分钟阅读量，对话风）
  • Instagram Caption（100字+10个hashtag）
  • Reddit Post（200字，零营销味，真诚讨论风）
  • 30秒动画脚本（3个变体，含视觉方向标注）

质量检查清单：核心信息跨格式一致、每种格式原生感、无营销腔、动画脚本有情绪落点。

Prompt 模板 Clarity Surgery — 30% 瘦身手术

Editor Agent 的核心工具。输入一段文本，输出精简30%的版本。

手术刀清单：

  • 砍掉废话填充词
  • 消灭被动语态
  • 把模糊词（perhaps, might, could, somewhat）全部替换
  • 抽象名词→具体动词
  • 超过20个词的句子→拆开
  • 杀掉不创造价值的副词

输出：不只给你结果，还告诉你修改前后字数对比 + 3个最重要的改动及原因。

Prompt 模板 Argument Builder — 说服力论述引擎

当你需要写观点文/说服性文章时使用。

8段式结构：

  • Hook（5%） — 惊人数据、具体故事或问题
  • Problem（15%） — 建立"哪里坏了"以及为什么重要
  • Current Solutions（15%） — 公平但批判地评估现有方案
  • Thesis（5%） — 一句话论点，不含糊
  • Evidence（35%） — 3-5个支撑点，每个含数据+专家意见+真实案例
  • Counterarguments（10%） — 正面回应最强的2个反对意见
  • Implications（10%） — 如果你对了，世界会怎样
  • Call to Action（5%） — 一个具体的行动步骤

核心原则："Steel-man the opposition"——用对手最强版本的论点来反驳，而非稻草人。

Prompt 模板 Story Overlay — 把无聊变精彩

当你有一堆正确但无聊的信息时，这个模板给它加上叙事骨架。

三幕结构：

  • ACT 1 — Setup（20%）：从读者认得的一个具体场景开始，引入主角，建立赌注
  • ACT 2 — Conflict（60%）：展示障碍和失败的尝试，构建"但是然后……"的转折，这里是所有事实内容的主战场
  • ACT 3 — Resolution（20%）：揭示解决方案，展示前后对比，以赢得感的行动建议收尾

规则：保留所有原始事实——添加叙事组织，不替换实质内容。故事服务于真相，不是反过来。

Prompt 模板 Empathy Rewrite — 技术内容人话翻译

把专业内容翻译成普通人能懂的语言。

对每个专业术语执行三步：

  • 用一句话定义它
  • 给一个日常生活的类比或具体例子
  • 解释为什么这对读者个人很重要

结构四问：这是什么？→ 我为什么该在乎？→ 它怎么工作的？→ 我下一步该干嘛？

终极测试："一个晚上10点已经累了的妈妈能不能看懂这段话，并且知道该做什么？"如果不行，再简化一轮。

Prompt 模板 Research Pipeline — 从资料到原创文章

当你有一堆研究资料/URL/摘录时，用这个模板把它们变成原创文章。

五步流程：

  • Extract — 从每个来源提取关键论点、数据、引用，标注出处
  • Find the Thread — 找到贯穿所有来源的叙事线，用一句话概括"所以呢？"
  • Add Original Analysis — 不只是摘要！为每个要点加上对读者的意义分析
  • Structure — 用最惊人的发现开头，3-5主要段落，段落间有过渡逻辑
  • Cite — 自然引用，0%直接抄袭，未验证的标记 [NEEDS VERIFICATION]
内容原则 写作红线与反面清单

Content Factory 有一份非常实用的写作反面清单，所有 Agent 都遵守：

✅ 必须做到：

  • 每篇内容只讲一个观点
  • 具体 > 模糊（"我们把生产时间缩短60%" vs "我们提高了效率"）
  • 展示，不要告诉
  • 用最有趣的东西开头

🚫 绝不使用的词：

  • "delve," "tapestry," "leverage," "harness," "utilize"
  • "excited to announce," "game-changer," "revolutionary," "disruptive"
  • "at the end of the day," "in today's fast-paced world," "now more than ever"
  • 任何可以套在任何公司身上的万能句

人味测试："一个真人会这样跟朋友说话吗？"——如果不会，重写。

高级用法 子 Agent 并行生产

Content Factory 支持用 sessions_spawn 并行启动多个 Agent，适合大批量内容生产：

// 启动 Writer 写长文
sessions_spawn --task "Act as the Writer agent. 
Write a 1,500-word article on [topic] for [audience]."

// Writer 完成后，同时启动 Remixer + Scriptwriter + Headline Machine
sessions_spawn --task "Act as the Remixer. 
Remix this article into: Twitter, LinkedIn, email, captions."

sessions_spawn --task "Act as the Scriptwriter. 
Write 3 variations of a 30-sec hook script."

sessions_spawn --task "Act as the Headline Machine.
Generate 20 headlines, sort by estimated CTR."

这样一篇文章从初稿到全平台内容，可以并行完成后半段工作。

👍 优点

  • 角色分工思路极好。这是我见过对"多 Agent 协作"最落地的设计之一。不是空谈"多智能体"概念，而是每个 Agent 都有明确的输入/输出/质量标准/Prompt 模板。拿来就能用。
  • Prompt 模板质量极高。尤其是 Argument Builder 的8段式结构和 Clarity Surgery 的手术刀清单，这些不只是给 AI 的 Prompt——即使人类编辑自己用，也是一套很好的写作方法论。
  • 写作反面清单太实用了。"delve/tapestry/leverage"这些 AI 味词汇黑名单、"人味测试"——这些规则让产出真的少一些 AI 味。
  • Remix Engine 的格式覆盖广。11种输出格式基本覆盖了主流内容分发渠道，Reddit post 那个"零营销味"的要求特别懂行。
  • 模块化设计灵活。全流水线或单个 Agent 都能用，不绑定。

👎 缺点

  • 明显面向英文写作市场。平台以 Twitter/LinkedIn/Instagram 为主，没有微信公众号、小红书、知乎、B站等中文平台适配。对中国创作者来说，至少一半输出格式需要重新设计。
  • 带有强烈的宗教/育儿垂类烙印。多个模板里出现"theological guardrails""Christian mom, 28-42""conditional love"等特定受众标签，这说明它最初是为一个宗教育儿品牌定制的，通用性打折扣。
  • 声称10个 Prompt 文件，实际缺失2个。headlines.md 和 voice-cloner.md 在 SKILL.md 中提到但并不存在，文件结构声明和实际不一致。$9 的付费产品不该有这种缺漏。
  • 没有实际的自动化脚本。所谓"多智能体"依赖用户手动用 Prompt 调度，没有真正的编排层。sub-agent spawn 只是一个调用建议，不是内置能力。
  • 缺乏效果评估闭环。生产出11种格式很酷，但没有任何机制追踪哪种格式表现好、哪些标题真的带来了点击。有生产没复盘。

💡 适用建议

  • 最适合：英文内容创作者，尤其是个人品牌/自媒体/教育类创作者。如果你每周写1篇深度内容然后分发到多个英文平台，这个 Skill 能帮你省几个小时。
  • 对 Rosetta 的价值：核心价值不在直接使用，而在学习它的设计思路。角色分工模式、Prompt 模板设计、写作反面清单——这些方法论可以迁移到中文内容生产场景。特别是百科项目可以参考它的 Editor Agent 五轮审稿流程来提升条目质量。
  • 改造建议：如果要做中文版，把 Remixer 的输出格式换成公众号/小红书/知乎/B站/飞书文档，把写作反面清单里加上中文 AI 味词汇（"赋能""沉淀""打造"等），就能变成一个很好的中文内容工厂。
