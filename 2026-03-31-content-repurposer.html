这个 Skill 是什么？

一句话：一篇博客进去，五个平台的内容出来，声音一致，直接能发。

Content Repurposer 解决内容创作者的"分发地狱"——你辛苦写了一篇长文，然后要花4小时手动改写成 Twitter 线程、LinkedIn 帖子、邮件通讯、Instagram 标题和 Threads 帖子。这个 Skill 用一条命令搞定一切，而且保持你的个人风格不走样。

为什么值得学？

在内容为王的时代，创作不是瓶颈，分发才是。写一篇好文章需要2-3小时，但把它适配到5个平台又要4+小时。Content Repurposer 把后半段自动化了。

  • 刚需 个人博主、自媒体人——一篇文章变一周的社交内容
  • 适用 出海营销团队——同时管理 Twitter/LinkedIn/Instagram 多平台
  • 适用 播客主、课程创作者——把文字稿/讲义变成推广素材
  • 学习价值 即使不用这个工具，它的平台适配规则本身就是一份社交媒体运营指南
核心理念：Create Once, Publish Everywhere

声音一致性引擎（Voice Profile）

这是 Repurposer 区别于简单模板工具的关键。你可以定义：

{ "tone": "professional-casual", "personality": ["direct", "insightful", "practical"], "avoid": ["corporate jargon", "excessive hype", "clickbait"], "signature_phrases": ["Here's the thing:", "The reality:"], "emoji_level": "moderate", "first_person": true }

不管是 Twitter 的短平快还是 LinkedIn 的商务风，都从这个 Profile 出发适配——你在所有平台上听起来都像"你"。

Prompt 工程亮点

每个平台脚本不是简单地告诉 AI "改写成XX"，而是构建了一套三层约束系统：

  • Voice 层：从 config.json 读取 tone/personality/avoid/emoji_level，确保人格一致
  • Platform 层：每个平台独有的长度限制、格式规则、Hook 策略、CTA 风格
  • User 层：品牌名、网站、主CTA——自动融入每个版本

还有个巧妙设计：Prompt 开头要求 AI 先做 Constraint Checklist & Confidence Score（约束清单自检打分），逼 AI 在动笔前先过一遍所有约束。

有料但不成熟

👍 优点

  • 解决真痛点。"一稿多发"是所有内容创作者的日常折磨。4小时改写 → 30秒自动生成，ROI 一目了然。这不是"有了更好"的工具，是"没有会死"的效率需求。
  • Voice Profile 设计精巧。不是简单的"改写成XX风格"，而是用 tone + personality + avoid + signature_phrases 四维定义声音，确保跨平台一致性。这个概念比大多数内容工具都想得深。
  • 平台知识密度高。每个平台的适配规则写得很细：LinkedIn 不在正文放链接、Instagram 前50字符是Hook、Twitter 线程只在末条放标签……这些是实打实的运营经验沉淀。
  • Prompt Engineering 示范。每个脚本里的三层约束（Voice + Platform + User）+ Confidence Score 自检机制，是很好的 Prompt 工程教学案例。
  • 配置驱动，不改代码。全部行为通过 JSON 控制，换个平台策略改个字段就行。

👎 缺点

  • 脚本全是 Mock，不能真跑。打开任何一个平台脚本，你会发现 LLM 调用部分写的是 MOCK_RESPONSE="..."。所有7个脚本都只是输出硬编码的示例文本。它本质上是一个精心设计的架构原型，不是可运行的产品。
  • 依赖不存在的 CLI。脚本里写的 clawdbot llm、clawdbot web_fetch 等命令根本不存在于当前 Skill 体系里。就像给一辆车画了完美的仪表盘，但没有装引擎。
  • 平台覆盖有局限。5个平台全是海外的——Twitter/LinkedIn/Instagram/Threads/Newsletter。对中文用户来说，微信公众号、小红书、微博、知乎才是主战场。零本土适配。
  • 没有质量反馈环。生成完就完了，没有 A/B Hook 测试、没有互动率预测、没有"这条推文预计表现"的评估。在 advanced 配置里有 engagement_score: false 和 generate_multiple_hooks: false，但实际没有实现。
  • 剪贴板功能只支持 macOS。用的是 pbcopy，Linux 和 Windows 用户直接无视。

💡 适用建议

当学习材料读，不要当工具用。

Content Repurposer 的真正价值不在于它能不能跑起来（不能），而在于它展示了一个内容分发自动化系统该怎么设计：

  • Voice Profile 的四维模型可以直接搬到你自己的 Prompt 里
  • 每个平台的适配规则是一份现成的社交媒体运营手册
  • 三层约束的 Prompt 结构（Voice + Platform + User）是通用的 Prompt 设计模式
  • config.json 驱动行为的架构思路可以用在任何 AI 工具项目中

如果你真的需要一稿多发的工具，建议看 Skill 市场的 Content Factory（#002期）——它至少有完整的 Prompt 模板和多Agent编排。或者直接把 Repurposer 的 Prompt 模式搬进 WorkBuddy 的对话里手动用。

--- name: content-repurposer description: Transform long-form content into platform-optimized snippets. description_zh: "将长文内容转化为多平台优化的社交媒体片段" version: 1.1.0 --- # Content Repurposer — Create Once, Publish Everywhere **Stop reformatting. Start publishing.** You wrote one great piece. Now you need it as a Twitter thread, LinkedIn post, newsletter section, and Instagram caption. That's 4+ hours of rewriting, reformatting, and maintaining voice consistency. Or... 30 seconds. Content Repurposer takes your long-form content (blog post, video transcript, podcast notes, article) and automatically generates platform-optimized versions. Same core message. Different formats. Your voice throughout. **What makes it different:** This isn't a template engine—it's intelligent adaptation. The skill understands what makes content perform on each platform: Twitter wants punchy hooks and thread flow, LinkedIn values professional insights and storytelling, newsletters need scannable sections and CTAs, Instagram demands visual hooks and emoji. ## The Problem Content creators face the **repurpose grind**: - ✍️ You create one killer blog post (2-3 hours) - 🔄 Manually reformat for Twitter (45 min) - 🔄 Adapt for LinkedIn (30 min) - 🔄 Write newsletter version (30 min) - 🔄 Craft Instagram caption (20 min) - 😤 Total: 4+ hours of reformatting, still inconsistent voice ## The Solution ```bash repurpose.sh blog-post.md # → twitter-thread.txt # → linkedin-post.txt # → newsletter.md # → instagram-caption.txt # → threads-post.txt (bonus!) ``` 30 seconds. Five platforms. Your voice. Ready to copy-paste and publish. ## Setup 1. Run `scripts/setup.sh` to initialize config 2. Edit `~/.config/content-repurposer/config.json` with your voice settings 3. Test with: `scripts/repurpose.sh examples/sample-post.md --dry-run` ## Scripts | Script | Purpose | |--------|---------| | `scripts/setup.sh` | Initialize config directory | | `scripts/repurpose.sh` | Main script: all platforms at once | | `scripts/twitter-thread.sh` | Twitter thread only | | `scripts/linkedin-post.sh` | LinkedIn post only | | `scripts/newsletter.sh` | Newsletter section only | | `scripts/instagram-caption.sh` | Instagram caption only | | `scripts/threads-post.sh` | Meta Threads post only | ## How It Works 1. **Parse Input**: Read long-form content (markdown, .txt, URL) 2. **Extract Core**: Identify main thesis, key points, quotes, stats 3. **Platform Adapt**: For each enabled platform: - Apply format rules (thread structure, char limits, emoji) - Maintain voice/tone from config - Add platform-specific hooks and CTAs - Optimize for engagement patterns 4. **Output**: Save to `output/` directory, optionally copy to clipboard ## Voice Consistency ```json "voice": { "tone": "professional-casual", "personality": ["direct", "insightful", "practical"], "avoid": ["corporate jargon", "hype", "clickbait"], "signature_phrases": ["Here's the thing:", "The reality:"], "emoji_level": "moderate" } ``` Every platform adaptation respects these settings. ## Platform Specs ### Twitter/X Threads - Length: 3-10 tweets, 280 chars/tweet - Hook: Bold opening (question, stat, or bold claim) - Best for: Hot takes, frameworks, step-by-step guides ### LinkedIn - Length: 1300-2000 characters - No links in body, short paragraphs - Best for: B2B insights, career lessons, thought leadership ### Email Newsletter - Length: 200-500 words per section - Scannable sections with headers - Best for: Deep dives, curated insights ### Instagram - Length: 150-300 characters - Emoji-heavy, visual language - Best for: Visual content tie-ins, quick tips ### Meta Threads - Length: 500 characters max - Casual, conversational - Best for: Quick insights ## Pro Tips 1. Subject Line First: Generate 5 subject line options 2. Hook Testing: Generate multiple hooks, pick the best 3. Engagement Checklist: Each version has a clear CTA? 4. Platform Priority: Start with best-performing platform 5. Batch Days: Repurpose a month of content in one session ## Use Cases - Bloggers: One post → a week of social content - Podcasters: Episode notes → promotional content - Course Creators: Transcripts → marketing snippets - Consultants: One insight → multi-platform thought leadership - Agencies: Scale content for clients ## What It's NOT - ❌ Not a content generator (you provide source material) - ❌ Not a scheduler (use Buffer/Hootsuite for posting) - ❌ Not image creation (text only) Your content deserves more reach. Your time deserves better use.
